      Search help

      Overview

      Search scans the tail of each log file (default 50 MB per file, configurable as SEARCH_MAX_BYTES_PER_FILE). Use Scope to search all configured directories, one directory, or the file selected in the sidebar.

      Press Search or Enter. Use Clear to return to the live view. The toolbar Severity filter applies to search server-side. Save queries with Save and reload them from Saved searches.

      A progress bar appears below the toolbar while search or file loads run.

      Log files sidebar

      With LOG_RECURSIVE=1, files at the log root and under common local folders (apt, auth, sysstat, …) appear under localhost with subfolder labels; remote host folders (often IP names) stay as separate groups. Groups start collapsed.

      • Click a group row — expand or collapse its log files
      • Double-click a group row — set search scope to that group (same as choosing it in Scope)
      • Click a log file — open it in the main pane
      • ⏱ before a filename — scheduled LLM analysis is configured (hover for next run; muted if disabled)
      • ● (green) — file is actively watched in the live tail
      • All files — combined live stream of failure lines from all watched plain files

      Right-click a readable file for Schedule analysis… (nightly / every 2 nights / weekly, with optional webhook or email alerts).

      File view and live tail

      • Time range — limits the initial load by parsed event time (see Timestamps and remote logs below); if nothing matches, log-intel shows the recent tail with a hint.
      • All (tail window) — best for quiet logs (e.g. mail.log) or when you want byte-tail without a time cut-off.
      • While you stay scrolled to the top (newest-first) or bottom (oldest-first), new lines are appended automatically on plain files.
      • Scroll away from the edge to read history without the view jumping.

      Timestamps and remote logs

      Logs forwarded by syslogpusher (or other remote senders) often have two timestamps on one line:

      • Leading prefix — when the central syslog server received the message (e.g. May 18 06:39:49 host …)
      • Embedded in the message — the real application event time (e.g. [sms.2024-11-05] - 23:32:55.288 in Pri.log)

      By default log-intel uses the leading timestamp. For known formats, timestamp parsers (Settings → Timestamp parsers) extract the embedded date instead — used for sort order, Time range filtering, and export.

      Built-in SMS Pri logs is enabled for Pri.log. Lines without a matching embedded date fall back to the leading timestamp. Other filenames are unaffected.

      On Windows hosts, enable Only new on the relevant syslogpusher directory watch to avoid replaying old file content on service restart.

      Search scope

      • All directories — search every readable file under all configured LOG_DIRS
      • Directory name — limit search to one log root
      • Host path — when recursive grouping is enabled, each host subdirectory appears here; limits search to files under that folder
      • Selected file — appears when a sidebar file is selected; searches only that file

      Text mode (boolean queries)

      Text mode matches substrings in each log line. Combine conditions with boolean operators, similar to Palo Alto log search:

      • AND — all conditions must match (also implied between adjacent terms: error ssh means error AND ssh)
      • OR — at least one condition must match
      • NOT — negates the following term or grouped expression
      • ( ) — group expressions to control precedence
      • "phrase" or 'phrase' — match an exact phrase containing spaces

      AND binds tighter than OR. Example: a OR b AND c is evaluated as a OR (b AND c).

      Text search is case-insensitive unless the server is configured with SEARCH_CASE_SENSITIVE=1.

      Text examples

      Failed AND ssh
      Lines containing both Failed and ssh.
      error AND (ssh OR sudo)
      Lines with error and either ssh or sudo.
      "failed password" AND NOT success
      Lines with the phrase failed password but not success.
      (deny OR denied) AND src
      Lines mentioning denial together with src.

      Regexp mode

      Regexp mode uses one Python re pattern per search. Boolean AND/OR operators are not applied in this mode — use regexp syntax instead, e.g. error|fail or (ssh|sudo).*denied.

      • Analyze entire log file (sidebar) — full-file analysis; large files use RAG + embeddings.
